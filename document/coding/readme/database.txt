項目の命名規則
	key		文字列
	id		シリアル(integer)
↑中間テーブル以外はどちらかが必須でprimary

2つ (+中間テーブル) 以上のテーブルを結合させるSQLは書かない。
(その時点で設計ミス)


文字列の扱い。

1. 一定数の候補から選択では、整数型, 列挙型, "char" を使う。

2. 255文字以内ならば varchar 型
メールアドレスがrfcで255までなので....
テキストボックス等。
定義は varchar 255にしておく。

3. それ以外は text 型
テキストエリア等

参考)http://lets.postgresql.jp/documents/technical/text-processing/1


検索時には、検索項目が複数のテーブルに跨らないようにDB設計をする
全文検索は、専用のテーブルを作った方が精度と速度が上がる。
